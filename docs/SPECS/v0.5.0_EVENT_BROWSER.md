# Sports Day Manager

## Version

v0.5.0

## Feature

Event Browser

---

# Goal

Introduce the Event Browser.

The organiser should be able to browse all configured events and select one ready for future result entry.

No scoring or event-specific logic is required in this release.

This release is purely about viewing and selecting events.

---

# User Story

As the Sports Day organiser,

I want to browse all available events,

so that I can select the next event to run.

---

# Functional Requirements

## Display Events

Display every enabled event returned by the existing API.

Each row should display:

- Event Name
- Event Format
- Points Profile
- Enabled status

---

## Select Event

Each event should be selectable.

When selected:

- highlight the selected row
- store the selected event in application state

No further action is required in this release.

---

## Event Details

Below the table display a simple summary card.

Display:

Event Name

Event Format

Points Profile

Enabled

Description (if available)

This card should update whenever another event is selected.

---

## Data Source

Use the existing backend API.

Do not modify the Apps Script backend.

Use:

Api.getEvents()

---

# Non Functional Requirements

Do not redesign the existing application.

Do not alter Competitor Management.

Do not modify Leaderboard functionality.

Preserve all existing behaviour.

Keep the architecture consistent with the current project.

---

# Architecture

Business logic remains in app.js.

Rendering should be placed in ui.js where appropriate.

The browser should not contain scoring logic.

No event engine should be introduced in this version.

---

# Acceptance Criteria

✓ Events load successfully

✓ Events display in a table

✓ Event selection works

✓ Selected event is highlighted

✓ Event summary updates correctly

✓ Existing pages continue to work

✓ No backend changes required

---

# Files Expected To Change

Likely:

web/js/app.js

web/js/ui.js

Possibly:

web/index.html

web/css/styles.css

No Apps Script files should require modification.

---

# Explicitly Out Of Scope

No editing events

No creating events

No deleting events

No scoring

No tournaments

No races

No leaderboard updates

No points calculation

These will be implemented in future releases.

---

# Definition of Done

The organiser can:

Open the Events page

↓

See every configured event

↓

Select an event

↓

View its details

Nothing more.
